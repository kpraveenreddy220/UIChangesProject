Overview

Various components of the SDK can be installed onto a Luxe terminal. This includes the SDK documentation and many of the development tools.

Luxe SDK Installation Package

The Luxe SDK Installation package includes:
  • A set of HTTP API Documentation pages linked to the main URL
  • A set of HTTP API test pages with Imbedded JavaScript linked to the documentation pages
  • A set of HTTP API functions that are required by PageBuilder and available for developers to use
  • A dedicated developer's account and home directory on the terminal
    • User name viking
    • Default password skeid
    • Home directory /home/viking
  • webdav is enabled to allow the developer's computer to browse / read / write files to the Luxe
After installing the SDK onto a Luxe terminal, there will be a number of HTTP API documentation and test pages on the Luxe's web server and some extra developer HTTP API's that are required for the operation of PageBuilder.

When the SDK package is installed onto the Luxe terminal, a soft link is created in the web server root /var/run/www/index.html which points to the SDK's main index page at location /home/sdk/html/index.html. This allow a developer to browse to the terminal's URL at http://URL and the terminal's web server will reply with the SDK main index page.

/home/sdk/bin files

This directory contains some "helper" functions that are required by some of the SDK functions.
Name Description
install.sh This is called by the cgi-bin package/install function to parse an uploaded developer's .TAR package file. It will untar the file and run the enclosed installPackage.sh BASH shell script. NOTE: As the installPackage.sh will run as user u_webserver in group gr_webserver, this user/group must have the necessary permissions to execute the commands in the installPackage.sh script.
mac This function calculates the MAC of a binary file using a supplied encryption key.

                    URL                                 Location              Description
http://URL/cgi-bin/sdk/file/dir /home/sdk/cgi-bin/file/dir This function returns a directory of files within the /home/sdk/files/projects area.
http://URL/cgi-bin/sdk/file/put /home/sdk/cgi-bin/file/put This is a C binary that implements an enhanced file/put command that is required for PageBuilder. It supports the POST'ing of files in multipart-form, application/json, and text/xml formats. The POST'd files are stored within the /home/sdk/files/ directory subsystem.

http://URL/cgi-bin/sdk/package_sdk/load /home/sdk/cgi-bin/package_sdk/load This is a special SDK version of the package/load HTTP API for software developers. This version can install non-signed software into the developer's terminal and it uses a simplified packaging format. The production package/load expects a signed .RPM format file which includes the pre an post installation instructions to be included as a SPEC file withing the .RPM.

The developer's packing format is a simple .TAR file that contains the files to be installed / upgraded, and a "special" file named installPackage.sh that contains the necessary pre and post installation instructions as a simple BASH script.
